<objective>
Add cached token usage display to the Token Usage page (NanobotTokenPanel).

Purpose: Claude Code session JSONL files contain `cache_read_input_tokens` and `cache_creation_input_tokens` fields that track Anthropic prompt caching. Currently, these values are folded into the combined `input_tokens` during sync and the breakdown is lost. This plan preserves the cache breakdown in SQLite, aggregates cache statistics, and adds cache hit rate, savings, and token breakdown cards to the Token Usage page.

Output: Token Usage page shows cache hit rate (%), estimated savings ($), cache read tokens, and cache creation tokens alongside existing stats.
</objective>
