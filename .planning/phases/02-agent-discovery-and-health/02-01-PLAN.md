    2. Create `src/lib/agent-health.ts` implementing:
       - `checkPortAlive(port, host?, timeoutMs?)`: Uses `net.Socket` TCP connect. Returns Promise<boolean>. Default host '127.0.0.1', timeout 2000ms.
       - `readLastLines(filePath, maxBytes?)`: Reads last N bytes of file using `fs.openSync` + `fs.readSync` with buffer positioned at `file.size - maxBytes`. Removes potentially partial first line. Default 4096 bytes. This avoids reading entire JSONL files into memory (RESEARCH pitfall #2).
       - `getLatestActivity(sessionsDir)`: Finds most recent .jsonl file by mtime, reads last lines, walks backward to find last assistant message (skipping _type: "metadata" lines per RESEARCH pitfall #5). Returns SessionActivity or null.
       - `detectErrors(workspacePath, agentName, since)`: Combines errors from two sources: (a) JSONL sessions -- scans last 16KB of each .jsonl file for tool errors and rate limits after `since` timestamp, (b) error log file at `{workspacePath}/logs/{agentName}-error.log` -- reads last 32KB, looks for Python traceback patterns (`/^\w+Error:/`, `/^\w+Exception:/`) and channel errors (`telegram.error.Conflict`). Returns AgentError[].
       - `getChannelStatuses(agent, portAlive, errors)`: Takes discovered agent, port check result, and error list. For each enabled channel: connected = portAlive (gateway running implies channels running), error = recent channel_error in errors list. Returns ChannelStatus[].
       - `computeCompositeHealth(data)`: Takes { portAlive, lastActivityMs, errorCount24h, criticalErrors, channelsDown, totalChannels }. Computes four HealthDimension values per the user's locked decisions: green (alive + no errors + channels OK + recent), yellow (alive but degraded), red (dead, critical errors, or unreachable). Worst dimension wins. Returns CompositeHealth.
       - `checkAgentHealth(agent)`: Orchestrates: port check, activity check, error detection, channel status, composite health. Returns AgentHealthSnapshot.

    IMPORTANT performance constraints from RESEARCH.md:
    - Read only last N bytes of JSONL files (not entire file -- can be 87KB+)
    - Read only last 32KB of error logs (can be 2.9MB+)
    - Error log may lack timestamps -- use file modification time as approximate reference
    - Critical errors = process dead OR all channels down. Non-critical = individual tool errors, single channel down.
