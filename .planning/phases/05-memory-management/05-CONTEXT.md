# Phase 5: Memory Management - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Browse and edit agent memory files (MEMORY.md, SOUL.md, IDENTITY.md, HISTORY.md) and subdirectories (episodes/, graph/, procedures/, topics/) from the dashboard. Operator+ role can edit files; viewer role can browse read-only. Existing Soul tab is merged into the new Memory tab. Changes persist to the agent's filesystem.

</domain>

<decisions>
## Implementation Decisions

### Browser location & layout
- Memory browser lives as a new "Memory" tab in the agent slide-out panel (alongside Overview, Errors, Channels, Lifecycle)
- Two-column layout within the tab: collapsible file tree (~200px left) + file viewer/editor (remaining right)
- Existing Soul tab merged into Memory tab — SOUL.md accessible via file tree. Remove separate Soul tab.
- "View Memory" quick-link added to the Overview tab (like Sessions has "View Sessions") — switches to Memory tab on click

### Markdown editor experience
- Plain textarea for editing, no rich editor dependencies (no CodeMirror/Monaco)
- Toggle mode: "Edit" and "Preview" buttons — one view at a time, not side-by-side
- Edit mode shows textarea; Preview mode shows rendered markdown via existing MarkdownRenderer component
- Manual save via "Save" button + Ctrl+S keyboard shortcut
- Unsaved changes indicator (dot on filename or tab)
- "Saved" toast notification on successful save
- Viewer role sees Memory tab in read-only mode — Edit button and Save button hidden, browse and read only

### File tree presentation
- Flat root files + expandable memory/ directory — mirrors actual filesystem structure
- Show all .md files and directories containing .md files. Non-.md files (logs, binaries, scripts) hidden.
- Full recursive depth — browse as deep as the filesystem goes, every directory expandable
- Last modified timestamp shown next to each file (relative format: "2h ago", "Mar 8")
- No file size display — just names and timestamps

### Edit safety & guardrails
- Save anyway with warning when agent is running: yellow banner "Agent is running — changes may be overwritten by the agent"
- Unsaved changes guard: warn "You have unsaved changes. Discard?" when navigating to another file or closing the tab
- Read-only files: HEARTBEAT.md and SESSION-STATE.md are non-editable (auto-generated by agent). Show lock icon + "Auto-generated by agent" label.
- All other .md files editable by operator+ role

### Claude's Discretion
- File tree expand/collapse animation and styling
- Exact textarea sizing and scroll behavior
- How to detect if agent is running (reuse health status from Zustand store)
- Path traversal safety implementation (extend existing resolveWithin pattern)
- How to handle binary or non-UTF8 files encountered during recursive scan
- API route structure (single route for all memory files vs separate routes)

</decisions>

## Specific Ideas

- Two-column layout within the existing slide-out: file tree left, content right. Slide-out is ~600px wide, works well.
- File tree should feel like a simple file explorer — expand folders, click files to view, nothing fancy.
- Preview mode reuses the same MarkdownRenderer that renders agent chat messages in the Sessions panel.
- The "agent is running" warning should use the health status already in Zustand (no extra API call needed).
- HEARTBEAT.md and SESSION-STATE.md get a lock icon to signal they're auto-generated and non-editable.
