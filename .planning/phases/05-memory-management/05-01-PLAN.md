    - scanWorkspace excludes non-.md files (logs, scripts, binaries)

    - scanWorkspace sorts: directories first, then files, alphabetical within each group

    - writeMemoryFile rejects writes to read-only files (HEARTBEAT.md, SESSION-STATE.md) by throwing
    - writeMemoryFile rejects path traversal attempts by throwing

    1. `MemoryFileNode` interface:
       ```typescript
       export interface MemoryFileNode {
         path: string       // relative to workspace root, e.g. "SOUL.md" or "memory/MEMORY.md"
         name: string       // filename only
         type: 'file' | 'directory'
         modified?: number  // Unix ms timestamp
         readOnly?: boolean // true for HEARTBEAT.md, SESSION-STATE.md
         children?: MemoryFileNode[]
       }
       ```

    2. `READ_ONLY_FILES` constant: `new Set(['HEARTBEAT.md', 'SESSION-STATE.md'])`

       - For .md files: include with stat.mtimeMs and readOnly flag
       - For directories: recurse, include only if children.length > 0
       - Sort: directories first, files second, alphabetical within group
       - Use `path.relative(workspacePath, fullPath)` for relative paths

    4. `readMemoryFile(workspacePath: string, filePath: string): { content: string; modified: number; readOnly: boolean } | null` -- uses `resolveWithin(workspacePath, filePath)` for safety, `readFileSync` for content, `statSync` for modified. Returns null if file doesn't exist. Wraps read in try/catch for non-UTF8 files.

  <action>
    Create two API route files following the established route handler pattern (requireRole + try/catch + logger.error).

    **Route 1: `src/app/api/agents/[id]/files/route.ts`**

    `GET /api/agents/{id}/files` -- Returns the file tree for a discovered agent.

    Implementation:
    1. `requireRole(request, 'viewer')` -- any authenticated user can browse
    2. `readLimiter(request)` -- rate limit
    3. Resolve agent: `healthMonitor.getSnapshot().find(a => a.id === agentId)`. If not found, return 404 with `{ error: 'Agent not found or not yet discovered' }`.
    4. Call `scanWorkspace(agent.agent.workspacePath)` from `@/lib/memory-files`
    5. Return `{ tree: MemoryFileNode[], agentId: string }`
    6. Wrap in try/catch, log errors with `logger.error({ err }, 'GET /api/agents/[id]/files error')`

    Note: Use `{ params }: { params: Promise<{ id: string }> }` signature (Next.js 16 async params pattern -- see soul/route.ts for reference).

    **Route 2: `src/app/api/agents/[id]/files/[...path]/route.ts`**

    `GET /api/agents/{id}/files/{path}` -- Returns file content.

    Implementation:
    1. `requireRole(request, 'viewer')`
    2. `readLimiter(request)`
    3. Resolve agent via healthMonitor (same as Route 1)
    4. Extract file path from catch-all params: `const filePath = resolvedParams.path.join('/')`
    5. Call `readMemoryFile(agent.agent.workspacePath, filePath)`
    6. If null, return 404 `{ error: 'File not found' }`
    7. Return `{ path: filePath, content: string, modified: number, readOnly: boolean }`
    8. Catch resolveWithin errors (path traversal): return 400 `{ error: 'Invalid file path' }`

    `PUT /api/agents/{id}/files/{path}` -- Saves file content.

    Implementation:
    1. `requireRole(request, 'operator')` -- RBAC gate for writes
    2. `mutationLimiter(request)` -- rate limit mutations
    3. Resolve agent via healthMonitor
    4. Extract file path from catch-all params
    5. Parse body: `const { content } = await request.json()`
    6. Validate: content must be a string. If not, return 400.
    7. Call `writeMemoryFile(agent.agent.workspacePath, filePath, content)`
    8. Return `{ success: true, path: filePath, modified: number }`
    9. Catch read-only errors: return 403 `{ error: 'File is read-only (auto-generated by agent)' }`
    10. Catch path traversal errors: return 400 `{ error: 'Invalid file path' }`

    Use `{ params }: { params: Promise<{ id: string; path: string[] }> }` for the catch-all route.

    Import pattern: follow existing codebase conventions (NextRequest/NextResponse from 'next/server', named imports from @/lib/*).
